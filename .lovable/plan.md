

## План: Виправлення 404 помилок на Vercel (мобільні пристрої)

---

### Проблема

На мобільних телефонах при переході на профіль чи інші сторінки виникає помилка **404: NOT_FOUND** від Vercel. Це трапляється тому, що:

1. Додаток використовує React Router (клієнтська маршрутизація)
2. При оновленні сторінки або прямому переході Vercel шукає файл на сервері
3. Файл `/profile/123` не існує як фізичний файл → 404

---

### Рішення

Створити файл `vercel.json` з налаштуванням **rewrites**, який буде перенаправляти всі запити на `index.html`, де React Router обробить маршрут.

---

### Зміни у файлах

#### Файл 1: `vercel.json` (НОВИЙ ФАЙЛ)

Створити в корені проекту:

```json
{
  "rewrites": [
    {
      "source": "/((?!api/).*)",
      "destination": "/index.html"
    }
  ]
}
```

Це означає:
- Будь-який URL, крім `/api/*`, буде перенаправлено на `index.html`
- React Router отримає URL і покаже правильну сторінку

---

### Візуалізація

```text
ЗАРАЗ (проблема):

Користувач → /profile/123 → Vercel → Шукає файл → Не знаходить → 404


ПІСЛЯ ВИПРАВЛЕННЯ:

Користувач → /profile/123 → Vercel → vercel.json → index.html → React Router → Profile компонент
```

---

### Після публікації

Після додавання `vercel.json` потрібно:
1. Опублікувати зміни (Publish)
2. Vercel автоматично підхопить нові налаштування
3. Перевірити на мобільному що 404 більше не виникає

### Очікуваний результат

1. Всі маршрути працюють на мобільних пристроях
2. Оновлення сторінки не викликає 404
3. Прямі посилання на профілі та інші сторінки працюють
4. Кнопка "назад" в браузері працює коректно
